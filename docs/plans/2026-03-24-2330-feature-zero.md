# Plan: Feature Zero — single-agent proof of life

## Goal

Validate the full ADK pipeline end-to-end: agent imports, API key works, `adk web` serves the dev UI, and the orchestrator responds to a wildfire risk query. This unblocks all downstream agent work.

## Current State

The scaffold commit already delivered:
- `agents/orchestrator/agent.py` with `fireshield_orchestrator` using `gemini-2.0-flash` + `google_search`
- `pyproject.toml` with `google-adk[a2a]` dependency
- 8 passing smoke tests (imports, model, tools)
- `.env.example` with required vars

## Remaining Work

### Step 1: Create `.env` from `.env.example`
- Copy `.env.example` to `.env`
- User must provide `GOOGLE_API_KEY` (from https://aistudio.google.com/apikey)
- Set `GOOGLE_GENAI_USE_VERTEXAI=FALSE`

### Step 2: Add Feature Zero integration test
- Add `test/test_feature_zero.py` that:
  - Verifies `.env` has `GOOGLE_API_KEY` set (skip if not)
  - Creates an ADK Runner with the orchestrator agent
  - Sends "Assess wildfire risk for a home in Tampa, FL"
  - Asserts a non-empty response is returned
- Mark as `@pytest.mark.integration` so smoke tests stay fast

### Step 3: Manual verification
- Run `uv run adk web agents/` and confirm UI loads at http://localhost:8000
- Send test query and confirm agent responds with wildfire risk content
- Verify `google_search` tool is invoked (visible in ADK UI)
