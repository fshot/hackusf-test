# Architecture Plan: FireShield AI

## Concept

FireShield AI is a multi-agent system that helps homeowners assess and improve their property's wildfire resilience. A homeowner describes their property and location; specialized agents assess vegetation risk, structural vulnerabilities, defensible space compliance, and generate a personalized hardening action plan — prioritized by cost, impact, and local fire code requirements. Privacy-first: no images stored, no data shared.

## Architecture

Hub-and-spoke topology with 4 ADK agents communicating via A2A protocol. Each agent is a separate Cloud Run service.

### Data Flow

1. Homeowner starts conversation with Orchestrator
2. Orchestrator gathers property details (location, age, materials, vegetation)
3. Orchestrator delegates to Risk Assessment → receives risk profile
4. Orchestrator delegates to Code Compliance → receives compliance gaps
5. Orchestrator delegates to Hardening Advisor → receives action plan
6. Orchestrator assembles FireShield Report with score, risks, actions, costs
7. Homeowner can ask follow-up questions (re-delegates as needed)

### Key Talking Points

- Multi-agent architecture: 4 specialized agents communicating via A2A protocol
- Privacy-first: no homeowner data persisted, no images stored
- Actionable output: not just "you're at risk" but "here's what to do and what it costs"
- Built on Google ADK + A2A — explored new technologies deeply
- Real-world impact: wildfire home loss is preventable with proper hardening
